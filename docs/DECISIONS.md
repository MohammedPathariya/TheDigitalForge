# Digital Forge Decision Log

## D001: Use separate benchmark and live-demo paths

**Status:** Accepted

The full benchmark runs locally or in CI and produces static result artifacts. The deployed application offers bounded individual runs. Recruiters should not have to wait for or fund a complete benchmark execution.

## D002: Keep evaluator tests independent

**Status:** Accepted

Hidden benchmark tests are maintained by the benchmark harness and never generated by the agents being evaluated. Agent-generated tests are development aids only.

## D003: Use two sandbox adapters

**Status:** Accepted

Docker is used locally and in CI to support the Docker sandbox claim. Modal Sandboxes are used by the free-tier hosted demo because Render Free is not a Docker host.

## D004: Separate algorithm and RAG evaluation

**Status:** Accepted

The 20-task algorithm suite measures baseline, first-shot, and self-healing performance. A smaller API-focused suite measures documentation retrieval and library hallucinations. Pure algorithm tasks are not a credible RAG evaluation.

## D005: Replace Streamlit with Next.js

**Status:** Accepted

The frontend will use Next.js and TypeScript. FastAPI will replace the minimal Flask interface to provide typed endpoints, background run state, polling, and cancellation.

## D006: Use one revamp branch for the sequential week

**Status:** Accepted

All seven phases will proceed sequentially on `mjp/revamp-digital-forge`, with focused commits for each phase. The branch will be reviewed and merged into `main` after end-to-end verification.

## D007: Measure before updating claims

**Status:** Accepted

Resume and LinkedIn percentages will be replaced with actual experiment results. Reports will include raw counts, model configuration, prompt versions, and task-level outcomes.

## D008: Declare an offline sandbox capability set

**Status:** Accepted

Docker and Modal use one pinned package set for network-isolated execution. The set covers
the versioned libraries represented in the bundled documentation corpus plus pytest and
HTTP testing support. Missing packages from this declared set are system configuration
failures; imports outside it remain application or generated-test responsibilities. The
pipeline does not install arbitrary packages during a run.
